## 一、打包前准备

### 1.1 确保Unity项目完整

在打包前，请确认以下内容：

✅ **必要文件检查**：

- [ ] 所有场景文件都已创建（MainMenu, LevelSelect, CharacterSelect, Battle）
- [ ] 所有C#脚本都已编写完成
- [ ] 必要的预制件（Prefabs）都已创建
- [ ] UI界面都已配置
- [ ] 项目无编译错误

✅ **资源文件检查**：

- [ ] 植物和僵尸的精灵图片（Sprites）
- [ ] 背景图片
- [ ] 音效文件（Audio）
- [ ] 字体文件（如需要特殊字体）

✅ **Unity版本确认**：

> ⚠️ **重要**：确认您使用的Unity版本
> - 推荐使用：**Unity 2022.3.x LTS** 或 **Tuanjie 5.0+**
> - 当前项目配置为Unity 2022.3.x
> - 较低版本可能存在兼容性问题

### 1.2 备份项目

> 💡 **强烈建议**：打包前先备份项目！

```bash
# 备份方法
1. 复制整个 PVZ_Arknights 文件夹
2. 保存到安全位置
3. 命名为 PVZ_Arknights_Backup_YYYYMMDD
```

### 1.3 测试游戏运行

在打包前，先在Unity编辑器中测试游戏：

1. 打开Unity编辑器
2. 打开 `Assets/Scenes/MainMenu.unity` 场景
3. 点击 **Play** 按钮 ▶️ 运行游戏
4. 确认游戏能正常运行
5. 如果有错误，先修复再打包

## 二、打包步骤详解

### 步骤1：打开Build Settings

在Unity菜单栏中：

```
File（文件） → Build Settings（生成设置）
```

快捷键：`Ctrl + Shift + B`（Windows）

### 步骤2：添加场景到构建列表

> ⚠️ **关键步骤**：必须将要打包的场景添加到列表中！

1. 在Build Settings窗口中
2. 确保 **Scenes In Build**（构建中的场景）列表包含：
   - ✅ MainMenu.unity
   - ✅ LevelSelect.unity
   - ✅ CharacterSelect.unity
   - ✅ Battle.unity

**添加方法**：
- 在Project面板中，找到 `Assets/Scenes/` 文件夹
- 拖拽场景文件到Build Settings窗口的Scenes In Build列表中
- 或者点击 **Add Open Scenes** 按钮

**列表顺序**：
> 💡 **重要**：第一个场景（通常MainMenu）会成为游戏启动时首先加载的场景
> - 推荐顺序：
>   1. MainMenu.unity（主菜单）
>   2. LevelSelect.unity（关卡选择）
>   3. CharacterSelect.unity（角色选择）
>   4. Battle.unity（战斗场景）

### 步骤3：选择目标平台

在Build Settings窗口中：

1. **Platform（平台）** 选择：
   - **PC, Mac & Linux Standalone**（独立平台）

2. **Target Platform（目标平台）**：
   - 选择 **Windows**

3. **Architecture（架构）**：
   - 选择 **x86_64**（64位，推荐）
   - 或选择 **x86**（32位，兼容性更好但性能稍差）

> 💡 **推荐**：选择 **x86_64**，现代电脑都是64位，兼容性很好

### 步骤4：配置Player Settings

点击 **Player Settings...** 按钮打开播放器设置：

#### 4.1 基本信息设置

在 **Inspector** 面板中找到 **Player** 设置：

**Product Name（产品名称）**：
```
PVZ_Arknights
```

**Company Name（公司名称）**：
```
YourCompany
```

**Default Icon（默认图标）**：
- 可以上传自己的图标（可选）
- 格式：PNG或ICO，推荐尺寸：256x256

#### 4.2 分辨率设置

展开 **Resolution and Presentation**：

**Default Is Native Resolution（默认使用原生分辨率）**：
- ✅ 勾选（推荐）

**Default Screen Width（默认屏幕宽度）**：
```
1920
```

**Default Screen Height（默认屏幕高度）**：
```
1080
```

**Fullscreen Mode（全屏模式）**：
- 选择 **Fullscreen Window**（全屏窗口）或 **Windowed**（窗口模式）

**Run In Background（后台运行）**：
- ✅ 勾选

#### 4.3 图标设置

在 **Icon** 部分：
- 可以设置不同尺寸的应用图标
- 至少设置一个256x256的主图标

### 步骤5：开始打包

返回 **Build Settings** 窗口：

1. 确认所有设置正确
2. 点击 **Build** 按钮开始构建

或者：

3. 点击 **Build And Run** 直接构建并运行

### 步骤6：选择输出位置

> ⚠️ **重要**：选择一个容易找到的位置！

**推荐位置**：
```
E:\clone\PVZ1\Build
```

**操作**：
1. 点击 **Build** 后
2. 在弹出的对话框中
3. 导航到 `E:\clone\PVZ1\`
4. 点击 **新建文件夹**，命名为 `Build`
5. 打开 Build 文件夹
6. 在"文件名"中输入 `PVZ_Arknights.exe`
7. 点击 **保存**

### 步骤7：等待打包完成

> ⏰ **耐心等待**：打包过程可能需要5-30分钟，取决于项目大小！

**打包中**：
- Unity会显示进度条
- 底部会显示当前正在处理的内容
- **不要关闭Unity或电脑！**

**打包完成**：
- 会看到提示："Build completed with X result(s)"
- 如果显示 **Succeeded**（成功），恭喜！
- 如果显示 **Failed**（失败），查看第五章常见问题解决

## 三、打包配置说明

### 3.1 Player Settings详细配置

#### Other Settings（其他设置）

**Rendering（渲染）**：
- Color Space：选择 **Gamma**（推荐）或 Linear

**Optimization（优化）**：
- Api Compatibility Level：选择 **.NET Framework** 或 **.NET Standard**

#### Publishing Settings（发布设置）

**Compression Method（压缩方式）**：
- 选择 **None**（不压缩）或 **LZHAM**（推荐）

### 3.2 Quality设置

在菜单中：
```
Edit → Project Settings → Quality
```

**建议**：
- 设置默认质量等级为 **Medium** 或 **High**
- 确保 **Anti-Aliasing** 设置为 **2x** 或更高

### 3.3 贴图压缩设置

```
Edit → Project Settings → Editor
```

**Texture Compression（纹理压缩）**：
- 选择 **DXT**（PC推荐）
- 或选择 **ASTC**（高质量）

## 四、打包后测试

打包完成后，导航到输出目录：

```
E:\clone\PVZ1\Build\

应该看到以下文件：

```

**测试步骤**：

1. **双击** `PVZ_Arknights.exe` 启动游戏
2. 观察游戏启动界面
3. 测试主菜单是否正常显示
4. 尝试点击各个按钮
5. 进入战斗场景测试

**检查项**：
- ✅ 游戏是否流畅运行（目标60FPS）

## 五、常见问题解决

### 问题1：Build失败 - 场景未添加

**错误信息**：
```
BuildFailedException: No scenes have been added to the build
```

**解决方法**：
1. 打开 **Build Settings**
2. 确保所有场景都添加到 **Scenes In Build** 列表
3. 至少添加一个场景

### 问题2：Build失败 - 脚本编译错误

**错误信息**：
```
Compilation failed
Assets/Scripts/XXX.cs(XX,XX): error CSXXXX: ...
```

**解决方法**：
1. 在Unity中打开Console面板（`Ctrl + Shift + C`）
2. 查看具体的编译错误
3. 修复所有脚本错误
4. 重新打包

### 问题3：Build成功但运行崩溃

**可能原因**：
- 缺少必要的资源文件
- 场景引用缺失
- 代码运行时错误

**解决方法**：
1. 检查Unity Console的所有警告和错误
2. 确保所有Prefabs和资源都正确引用
3. 在编辑器中运行测试，修复所有问题
4. 重新打包

### 问题4：游戏运行但没有显示内容

**可能原因**：
- 相机设置问题
- Canvas配置问题
- 场景未正确加载

**解决方法**：
1. 检查MainMenu场景的Canvas设置
2. 确保Camera正确配置
3. 检查UI元素是否正确绑定

### 问题5：打包后的exe文件很大

**优化方法**：
1. 压缩纹理资源
2. 删除不必要的资源
3. 使用 **Build Report** 检查大文件
4. 优化资源导入设置

### 问题6：运行时提示缺少DLL

**错误信息**：
```
The file or directory XXXX.dll is missing
```

**解决方法**：
1. 重新打包
2. 确保完整的Build文件夹一起分发
3. 不要只复制exe文件

### 问题7：分辨率不适配

**解决方法**：
1. 在Player Settings中设置正确的分辨率
2. 勾选 **Default Is Native Resolution**
3. 或设置为窗口模式运行

## 六、高级打包选项

### 6.1 IL2CPP vs Mono

**IL2CPP（推荐）**：
- 性能更好
- 代码更难被反编译
- 构建时间较长
- 支持AOT编译

**Mono**：
- 构建速度快
- 支持动态代码（但游戏不建议使用）
- 兼容性稍差

**设置位置**：
```
Player Settings → Other Settings → Scripting Backend
```

### 6.2 符号文件（Symbol）导出

用于调试崩溃问题：

```
Player Settings → Publishing Settings → Debugging
勾选：Development Build
```

> ⚠️ 注意：发布版本不要勾选此项！

### 6.3 自定义启动画面

```
Player Settings → Splash Image
```

- 可以设置游戏启动时的Logo
- 建议尺寸：1920x1080 PNG格式

### 6.4 代码加密（可选）

可以使用第三方工具对DLL进行加密，提高安全性。

### 6.5 自动更新功能（可选）

可以使用Unity的Asset Store中的工具实现自动更新功能。
